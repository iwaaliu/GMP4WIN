What's New...

The CJLibrary now supports both Visual C++ 5 as well as Visual C++ 6. Enhanced code encapsulates the functionality of the new MFC classes introduced with VC 6, and extends them to VC 5 users! Several new controls have been added to the CLibrary since the release of 6.07. Here is an overview of the classes that are found in the CJLibrary 6.08 release...

  • CCJBrowseEdit - A CEdit derived class which has a push button associated with it. It can be used to search for directories, files or activate a popup menu.


  • CCJBrowseButton - A CButton derived class which is used by the CCJBrowseEdit class.


  • CCJCaption - A a CStatic derived class which is simular to the caption or info bars that are seen in Microsoft Outlook.


  • CCJCaptionButton - A CCJFlatButton derived class which is used by the CCJCaption class to activate a CCJCaptionPopupWnd window.


  • CCJCaptionPopupWnd - A CWnd derived class which is used by the CCJCaption class to display a popup child window simular to Outlook.


  • CCJColorPicker - A CButton derived class which displays a color selection popup window.


  • CCJColorPopup - A CWnd derived class which is activated by the CCJColorPicker class.


  • CCJComboBoxEx - Extends the combo box control by providing support for image lists. By using CCJComboBoxEx to create combo box controls, you no longer need to implement your own image drawing code. Instead, use CCJComboBoxEx to access images from an image list.


  • CCJControlBar - A CControlBar derived class which is simular to the sizing / docking windows that are seen in Developer Studio.


  • CCJCBarButton - A CCJFlatButton derived class which is used by the CCJControlBar class.


  • CCJDateTimeCtrl - Encapsulates the functionality of a date and time picker control. The date and time picker control (DTP control) provides a simple interface to exchange date and time information with a user. This interface contains fields, each of which displays a part of the date and time information stored in the control. The user can change the information stored in the control by changing the content of the string in a given field. The user can move from field to field using the mouse or the keyboard.


  • CCJDockBar - A CDockBar derived class used by CCJFrameWnd and CCJMDIFrameWnd to offset CCJToolBars to allow 3D borders.


  • CCJDockContext - A CDockContext derived class which is used by CCJControlBar and CCJMiniDockFrameWnd for docking / sizing control bars.


  • CCJExplorerBar - A CCJControlBar derived class which is simular to the explorer bar that is seen in Windows or Internet Explorer.


  • CCJFlatButton - A CButton derived class used to create flat buttons. This class can only be used with the BS_OWNERDRAW style bit set.


  • CCJFlatComboBox - A CComboBox derived class which is simular to the flat combobox as seen in the Microsoft office products.


  • CCJFlatHeaderCtrl - A CHeaderCtrl derived class which is simular to the listbox flat header controls seen in Visual Studio and Outlook.


  • CCJFlatSplitterWnd - A CSplitterWnd derived class which adds the ability to hide and show splitter panes, based upon its index, and to draw flat splitter windows simular to Oultook.


  • CCJFlatTabCtrl - A CWnd derived class which is used to create a flat tab control simular to DevStudio.


  • CCJMetaFileButton - A CButton derived class which is used by the CCJFlatTabCtrl class.


  • CCJFrameWnd - A CFrameWnd derived class which is used to reposition toolbar layout for 3D effect, in addition to docking methods for CCJToolBar and CCJControlBar class.


  • CCJMDIFrameWnd - A CMDIFrameWnd derived class which is used to reposition toolbar layout for 3D effect, in addition to docking methods for CCJToolBar and CCJControlBar class.


  • CCJMDIChildWnd - Provides the functionality of a Windows multiple document interface (MDI) child window, along with members for managing the window. This class extends the standard CMDIChildWnd class to allow CCJControlBar and CCJToolBar docking.


  • CCJHtmlView - Provides the functionality of the WebBrowser control within the context of MFC's document/view architecture. The WebBrowser control is a window in which the user can browse sites on the World Wide Web, as well as folders in the local file system and on a network. The WebBrowser control supports hyperlinking, Uniform Resource Locator (URL) navigation, and maintains a history list.


  • CCJHyperLink - A CStatic derived classc will open the default browser with the given URL when the user clicks on the link.


  • CCJListCtrl - A CListCtrl derived class which implements a flat header and generic sorting routines.


  • CCJListView - A CListView derived class which implements a flat header and generic sorting routines.


  • CCJMaskEdit - A CEdit derived class which allows text masking to be applied to the control to format it for special editing restrictions.


  • CCJDateEdit - A CCJMaskEdit derived class which is specifically geared toward editing date fields.


  • CCJTimeEdit - A CCJDateEdit derived class which is specifically geared toward editing time fields.


  • CCJMenu - A CMenu derived class which implements "cool" menus with icons.


  • CCJMenuData - Used by the CCJMenu class to define a single menu item.


  • CCJMenuBar - Implements an Office style menu bar. Currently only implemented for SDI interface.


  • CCJMenuBarFrameHook - Used by CCJMenuBar to intercept messages on behalf of its owning frame, as well as the MDI client window.


  • CCJMiniDockFrameWnd - A CMiniDockFrameWnd class which is used to handle docking for the CCJControlBar class.


  • CCJMonthCalCtrl - Encapsulates the functionality of a month calendar control. The month calendar control provides the user with a simple calendar interface, from which the user can select a date.


  • CJOutlookBar - A CListBox derived class which simulates a control simular to the Outlook bar as seen in Outlook, can only be used with the LBS_OWNERDRAWVARIABLE style bit set. very generic.


  • CCJPagerCtrl - A CWnd derived class which wraps the windows Pager common control.


  • CCJReBar - A control bar that provides layout, persistence, and state information for rebar controls.


  • CCJReBarCtrl - Encapsulates the functionality of a rebar control, which is a container for a child window. The application in which the rebar control resides assigns the child window contained by the rebar control to the rebar band. The child window is usually another common control.


  • CCJShell - Used to perform shell file operations.


  • CCJShellList - A CCJListCtrl derived class which is used to display system files and folders, typically associated with a CCJShellTree object.


  • CCJShellTree - A CTreeCtrl derived class which is used to display system files and folders, typically associated with a CCJShellList object.


  • CCJSizeDockBar - A CDockBar derived class which is used by the framework to handle CCJControlBar's.


  • CCJSortClass - This class will sort a List control by a column of text, integer, float or date/time type. It could be easily extended for other data types.


  • CCJTabCtrl - A CTabCtrl derived class which is used by CCJTabCtrl to help fix problems with repainting in the tab control.


  • CCJTabCtrlBar - A CCJControl bar derived class which implements a tabbed window simular to VisualStudio's workspace window.


  • CCJTabView - A CCtrlView derived class which can be used to create a view that contains a tab control.


  • CCJToolBar - A CCJToolBarBase derived class which Allows the creation of toolbars simular to VisualStudio, Microsoft Office or Internet Explorer.


  • CCJToolBarBase - A CControlBar derived class whis is used by the CCJToolBar class.


  • CCJToolBarCtrl - Provides the functionality of the Windows toolbar common control. This control (and therefore the CToolBarCtrl class) is available only to programs running under Windows 95 and Windows NT version 3.51 and later.


  • CToolCmdUI - CCJToolBar idle update through CToolCmdUI class


  • CCJWindowPlacement - A WINDOWPLACEMENT derived class which is used to save and restore your applications window position.


  • CContentItems - A utility class used as a Container for CJOutlookBar menu items.


  • CGfxGroupEdit - A CEdit derived class used by CGfxOutBarCtl for editing menu labels.


  • CGfxOutBarCtrl - A CWnd derived class which implements a folder bar control simular to Outlook.


  • CGfxSplitterWnd - A CSplitterWnd derived class used by CGfxOutBarCtrl.


  • CSubclassWnd - Generic class to hook messages on behalf of a CWnd. Once hooked, all messages go to CSubclassWnd::WindowProc before going to the window. Specific subclasses can trap messages and do something.


  • CSubclassWndMap - The message hook map is derived from CMapPtrToPtr, which associates a pointer with another pointer. It maps an HWND to a CSubclassWnd, like the way MFC's internal maps map HWND's to CWnd's. The first CSubclassWnd attached to a window is stored in the map; all other CSubclassWnd's for that window are then chained via CSubclassWnd::m_pNext.
